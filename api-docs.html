Введение

REST API для интеграции с платформой KOL Studio

KOL Studio API предоставляет программный доступ к данным создателей, контенту, подпискам и платёжным операциям. API использует стандарт REST, принимает и возвращает данные в формате JSON, работает исключительно по протоколу HTTPS.

ℹ Все запросы должны содержать заголовок Content-Type: application/json. Пустой body при POST-запросах допускается, но заголовок обязателен.

Быстрый старт

Выполните первый запрос за 30 секунд — получите список создателей:

Shell — cURL 
# Получить список создателей
curl -X GET "https://api.kolstudio.com/v1/creators" \
  -H "Authorization: Bearer kol_live_your_api_key" \
  -H "Content-Type: application/json" \
  -G \
  --data-urlencode "limit=10" \
  --data-urlencode "platform=instagram"

Аутентификация

Bearer-токен и OAuth 2.0

KOL Studio API поддерживает два способа аутентификации: API ключ (Bearer-токен) для серверной интеграции и OAuth 2.0 для работы от имени пользователей.

Bearer-токен

Передавайте API ключ в заголовке Authorization каждого запроса:

HTTP Header
Authorization: Bearer kol_live_your_api_key_here
Content-Type: application/json
Accept: application/json

API ключи Тариф: Partner

Верификация подписи

Каждый вебхук содержит заголовок X-KOL-Signature — HMAC-SHA256 от тела запроса с вашим секретом.

Python — Flask
import hmac, hashlib
from flask import request, abort

WEBHOOK_SECRET = "whsec_your_webhook_secret"

def verify_signature(payload: bytes, signature: str) -> bool:
    expected = hmac.new(
        WEBHOOK_SECRET.encode(),
        payload,
        hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(expected, signature)

# В обработчике вебхука:
@app.route("/webhook", methods=["POST"])
def webhook():
    sig = request.headers.get("X-KOL-Signature")
    if not verify_signature(request.data, sig):
        abort(400)
    event = request.json
    # Обработка события...
    return "", 200
Node.js — Express
const crypto = require('crypto');

const WEBHOOK_SECRET = 'whsec_your_webhook_secret';

function verifySignature(payload, signature) {
  const expected = crypto
    .createHmac('sha256', WEBHOOK_SECRET)
    .update(payload)
    .digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(expected), Buffer.from(signature)
  );
}

// Express обработчик:
app.post('/webhook', express.raw({ type: '*/*' }), (req, res) => {
  const sig = req.headers['x-kol-signature'];
  if (!verifySignature(req.body, sig)) {
    return res.status(400).send('Invalid signature');
  }
  const event = JSON.parse(req.body);
  // Обработка события...
  res.status(200).send('OK');
});

Статус коды

Стандартные HTTP-ответы API

КодСтатусОписание
200 OK Запрос успешно выполнен. Данные в теле ответа.
201 Created Ресурс успешно создан. Возвращается при POST-запросах.
400 Bad Request Некорректный запрос. Проверьте структуру и типы параметров.
401 Unauthorized Отсутствует или неверный API ключ.
403 Forbidden Недостаточно прав для выполнения операции.
404 Not Found Ресурс не найден. Проверьте правильность ID.
422 Unprocessable Entity Данные прошли синтаксическую проверку, но содержат ошибки валидации.
429 Too Many Requests Превышен лимит запросов. Используйте заголовок Retry-After.
500 Internal Server Error Ошибка на стороне сервера. Повторите запрос позже.
